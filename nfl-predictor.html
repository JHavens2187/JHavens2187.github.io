Executive Summary

This NFL Prediction System is a professional-grade analytics platform designed to identify inefficiencies in betting lines. Unlike traditional prediction models, it compares a "Vegas-Aware" baseline to a "Pure Football" ensemble to detect true value edges.

Data Architecture & Ingestion

  • Core Data: nflreadpy library via nflverse ecosystem (2002-2024)
  • Granularity: Play-by-play EPA, WPA, and participation reports
  • Environmental: Stadium Surface (Turf/Grass), Roof Status, Wind/Temp
  • Officials: Referee crew assignments included to detect officiating bias

Using EPA instead of box scores allows the system to judge the context-sensitive value of plays, improving predictive quality.

Feature Engineering ("The Brain")

  • Rolling averages for key metrics (3, 5, 8 games) to capture "Form"
  • Matchup differentials: Offense vs Defense efficiency comparison
  • Roster Health: 'key_players_out' and 'injury_advantage' variables
  • Rest differential: Calculation of 'rest_days' skew (e.g. Bye Week vs Short Week)
  • Standardization via StandardScaler

Rolling windows capture hot streaks while smoothing out random noise, providing a responsive measure of team form.

Modeling Philosophy: Dual-Key Protocol

The system operates two parallel tracks:

  • Track A: Vegas-Aware Baseline – Logistic Regression including Vegas spread (67.3% Accuracy)
  • Track B: No-Vegas Ensemble – Pure football metrics ensemble:
    • Optimization Strategy: Brute-force Grid Search used to determine optimal voting weights.
    • XGBoost (Weight: 2): Gradient Boosting to handle non-linear decision trees.
    • SVM (Weight: 14): Support Vector Machine (RBF kernel) identified as the stabilizer for defensive variance.
    • Neural Network (Weight: 8): MLP (64->32 neurons) captures abstract deep-learning patterns.
  • Weighted ensemble final accuracy: 59.93%

The ensemble uses weighted voting to balance strengths and weaknesses of individual models, smoothing over their biases.

War Room: Strategy & Execution

  • Convert Vegas spreads to implied win probabilities
  • Edge calculation: Edge = Model Probability – Vegas Probability
  • Money management: Quarter Kelly criterion proportional to edge
  • Streamlit dashboard: Live data fetching, edge highlighting, bankroll simulation

def get_ensemble_prediction(matchup_data):
    # Retrieve individual model probabilities
    p_xgb = model_xgb.predict_proba(matchup_data)[0][1]
    p_svm = model_svm.predict_proba(matchup_data)[0][1]
    p_nn  = model_nn.predict_proba(matchup_data)[0][1]
    
    # Apply Grid-Search Optimized Weights
    # Weights: SVM(14) > NN(8) > XGB(2)
    weighted_score = (p_xgb * 2) + (p_svm * 14) + (p_nn * 8)
    final_prob = weighted_score / 24.0
    
    return final_prob
